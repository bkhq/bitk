# UI-002 Suppress queue-operation/progress raw text in chat

- **priority**: P2
- **status**: completed
- **owner**: claude
- **created**: 2026-03-09

## Problem

`queue-operation`, `progress`, and `last-prompt` message types from Claude Code stdout are not handled in the normalizer's `parse()` switch. They fall through to `parseUnknown()` which extracts `data.content` (raw XML like `<task-notification>...`) and renders it as plain text in the chat UI.

## Solution

Add these types to the normalizer's `parse()` switch to return `null` (suppress). They are internal to Claude Code's lifecycle and have no user-facing value. Already suppressed in transcript-fallback.
