# UI-003 Remove devMode feature entirely

- **priority**: P2
- **status**: completed
- **owner**: claude
- **created**: 2026-03-09

## Problem

The per-issue `devMode` toggle is dead weight — `isVisibleForMode` now applies proper entry-type filtering regardless of mode. The only remaining effect of `devMode=true` is bypassing all filtering (showing token-usage, loading, init, hook_response), which is not useful in production.

## Solution

Remove `devMode` from: shared types, DB schema (migration), visibility utils, query layers, engine, routes, orchestration/lifecycle, SSE events, message rebuilder, frontend toggle, and i18n keys.
