# BUG-015 创建任务工作树默认关闭

- **status**: completed
- **priority**: P2
- **owner**: codex
- **created**: 2026-03-18

## 描述

创建任务弹窗当前会在 Git 仓库项目里默认开启工作树。需求是将默认行为改为关闭，仅在用户主动打开时才提交 `useWorktree=true`。

## 调查结果

- `apps/frontend/src/components/kanban/CreateIssueDialog.tsx` 当前使用 `useState(true)` 初始化 `useWorktree`
- 同文件还会在 `projectIsGitRepo` 变化时自动将其同步为开启
- 后端创建接口本身已在未传值时默认使用 `false`
