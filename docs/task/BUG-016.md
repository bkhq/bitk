# BUG-016 全量修复仓库现有 tsc 错误

- **status**: completed
- **priority**: P1
- **owner**: codex
- **created**: 2026-03-18

## 描述

用户要求修复仓库当前所有 `tsc --noEmit` 错误，目标是让前端与后端 TypeScript typecheck 均恢复通过。

## 调查结果

- `bunx tsc -p apps/frontend/tsconfig.json --noEmit` 当前主要失败在 Atlaskit pragmatic drag-and-drop 相关模块缺失，集中在 `KanbanBoard.tsx`、`KanbanCard.tsx`、`KanbanColumn.tsx` 和 `HomePage.tsx`，并伴随一批由缺失声明引起的隐式 `any`。
- `bunx tsc -p apps/api/tsconfig.json --noEmit` 当前同时存在外部 SDK 类型缺失与仓库内类型收紧后的回归错误。
- API 侧错误可分为几类：
  - `@agentclientprotocol/sdk` 无法解析，影响 ACP executor 相关文件。
  - Drizzle 布尔/数值列比较、insert 参数类型与返回值推断不匹配，影响 `db/helpers.ts`、`pending-messages.ts`、若干路由与测试。
  - 若干业务文件存在严格空值与联合类型收窄问题，如 `claude/normalizer.ts`、`issue/debug-log.ts`、`issue/queries.ts`、`message-rebuilder.ts`、`projects.ts`、`terminal.ts`。
  - 测试文件中存在对联合返回类型直接取字段、以及测试桩类型不匹配的问题。
- 该任务会跨越前端、后端、共享类型与测试，属于非平凡改动，需要单独计划跟踪。
