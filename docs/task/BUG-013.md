# BUG-013 原生 Claude probe 改为本地凭据判断

- **status**: completed
- **priority**: P1
- **owner**: codex
- **created**: 2026-03-18

## 描述

当前原生 `claude-code` 引擎的 `getAvailability()` 在本地二进制与版本检查之后，还会通过 `claude -p --output-format text --max-turns 1 --no-chrome -- hi` 发起一次真实会话来判断认证状态。该路径会触发 hooks、插件刷新、skills 注入与实际 API 请求，容易因为网络抖动、服务过载或插件开销在 15 秒 probe 超时内失败，最终把已安装且本地已有凭据的 Claude 误判为不可用。

## 调查结果

- `/app/bkd/data/logs/bkd.log` 多次记录 `probe_engine_failed`，错误为 `claude-code timed out after 15000ms`。
- `claude --version` 本地可快速返回，但真实探测命令会稳定卡住直到超时。
- Claude 自身 debug 日志显示该探测命令会进入完整会话初始化，并触发 `overloaded_error` 与重试。

## 实施范围

- 将原生 `claude-code` 的 auth 判断改为本地凭据检查。
- 保留现有二进制存在性与版本检查。
- 不修改 ACP 的 `acp:claude:*` 探测逻辑。

## 验证

- [x] `bunx eslint apps/api/src/engines/executors/claude/executor.ts`
- [x] 本地确认 `claude --version` 正常，auth 状态可由本地凭据直接判定为 `authenticated`
