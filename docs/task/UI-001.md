# UI-001 Fix chat UI: visibility, grouping, collapsible tool groups

- **status**: completed
- **priority**: P1
- **owner**: claude
- **createdAt**: 2026-03-09
- **plan**: PLAN-007

## Problem

1. **Visibility filter (BUG-004 regression)**: `isVisibleForMode()` in non-dev mode only allows `user-message` and `assistant-message`. All tool-use (including TodoWrite/TaskPlan), thinking, system-message, and error-message are completely invisible — both via HTTP API and SSE stream. The CHAT-001 UI redesign that added tool groups, task plans, and thinking rendering is effectively dead in non-dev mode.

2. **Broken tool grouping**: `task_progress` system messages call `flushToolBuffer()`, splitting consecutive tool calls into individual 1-item groups (e.g. "1 读取文件" x6 instead of "6 读取文件")

3. **Raw "task_progress" text**: normalizer falls through to default case, setting `content = data.subtype = 'task_progress'`

4. **No collapse**: `ToolGroupMessage` always shows all items with no toggle

## Expected

- Non-dev mode shows tool groups, task plans, thinking, errors (same as dev mode minus meta-turns)
- Consecutive tool calls group into a single tool group
- `task_progress` entries suppressed or silently skipped
- Tool groups collapsible with `<details>` element, default open
