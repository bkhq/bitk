# BUG-014 删除原生 Claude stdout transcript fallback

- **status**: completed
- **priority**: P1
- **owner**: codex
- **plan**: PLAN-024
- **created**: 2026-03-18

## 描述

原生 `claude-code` 当前仍保留一套 “stdout pipe 提前结束后，转而从 transcript JSONL 补读日志” 的 fallback 机制。用户确认该能力已不再需要，应删除相关解析与状态分支，避免继续维护一条已废弃的恢复路径。

## 调查结果

- fallback 实现在 `apps/api/src/engines/executors/claude/transcript-fallback.ts`。
- 运行时接入点在 `apps/api/src/engines/issue/process/register.ts`，stdout 异常结束时会置 `stdoutBroken` 并轮询 transcript。
- `apps/api/src/engines/issue/gc.ts` 会对 `stdoutBroken && engineType === 'claude-code'` 做特殊跳过。
- `apps/api/src/routes/processes.ts` 还会向调试接口暴露 transcript 路径。
