# WT-001 重构 Worktree 系统

- **status**: completed
- **priority**: P1
- **owner**: claude
- **created**: 2026-03-02
- **plan**: PLAN-005

## 描述

对 worktree 系统进行全面重构，解决 5 个核心问题：
1. Worktree 目录位置不合理（当前在仓库内部）
2. 缺少持久化 worktree 路径的数据库字段
3. 创建任务默认未启用 worktree
4. 状态转换时缺少 worktree 的清理/重建逻辑
5. 文件变更跟踪未考虑 worktree 目录

## 验收标准

- [x] Worktree 创建在 `data/worktrees/:projectId/:issueId/` 下
- [x] issues 表新增 `use_worktree` 布尔字段（路径由 `resolveWorktreePath` 确定性解析，无需持久化）
- [ ] 创建 issue 默认 `useWorktree = true`（当前默认 false，用户手动开启）
- [x] working 状态时重建 worktree；done 状态由后台 worktree-cleanup job 延迟清理
- [x] 文件变更 API 使用 worktree 目录
