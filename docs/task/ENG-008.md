# ENG-008 重构事件引擎：统一事件总线

- status: completed
- priority: P1
- owner: claude
- createdAt: 2026-03-03 03:00 UTC
- updatedAt: 2026-03-03 04:00 UTC

## Summary

全面重构后端事件系统，将现有 3 套独立的 pub/sub 实现（issue-events、changes-summary、engine events）统一为单一类型安全的事件总线。在 `@bitk/shared` 中定义 SSE 事件类型供前后端共用，集中化分散在 7+ 文件中的 `emitIssueUpdated` 调用，使消息处理逻辑可统一挂接，减少代码重复，提高可审计性。

## Scope

- 在 `@bitk/shared` 新增 SSE 事件类型定义
- 后端创建统一 `AppEventBus`，替换 3 套 pub/sub
- 集中化所有 `emitIssueUpdated` 调用点
- 简化 SSE route 为纯转发层
- 前端 EventBus 使用共享类型
- DevMode 过滤逻辑从 emit 函数提取为 middleware
- 迁移所有调用方（~15-20 文件）

## Acceptance Criteria

- 所有 SSE 事件类型在 `@bitk/shared` 中定义，前后端共用
- 后端只有一个 `AppEventBus` 实例，所有事件通过它收发
- 不再有分散的 `emitIssueUpdated` 直接调用
- SSE route 代码大幅简化
- 前端 EventBus 使用共享类型做编译时校验
- 所有现有测试通过
- SSE 实时推送行为不变（事件名、payload 格式兼容）
